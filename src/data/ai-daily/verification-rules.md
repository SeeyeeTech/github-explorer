# 事实核验铁律（旗舰版 · 两篇 skill 共引）

目标：杜绝 AI 幻觉 / slop，确保日报每个事实可追溯到一手源。参数见 `scoring.yaml > verify`。

## 1. 高分必须交叉验证

评分 ≥ `verify.verify_floor`（默认 7.0）的条目，必须有 ≥ `verify.min_independent_sources`（默认 2）
个**独立**信源交叉确认（`secondary_sources` 不得来自同一媒体集团/同一转载链）。
由并行「验证 SubAgent」对抗式核查（默认假设为假，去找反证）：
- 通过 → 保留评分。
- 不通过 → 封顶 5.9（降级为简讯）或丢弃。

## 2. 禁止凭记忆 / 外推数字

- `metrics[].value` 中每个具体数字、技术细节必须能回指 `source_url`。
- 不得从记忆「补全」「估算」「外推」任何数字。
- 条目自检字段 `numbers_from_source=false` 的数字一律剔除，不写进日报。
- 拿不准的指标宁可不写，也不编。

## 3. 事件日期核实

- `event_date` 必须是**真实事件发生日期**，不是页面可访问日期 / 抓取日期。
- 通过原始公告/官方博客核实；核实失败则不计入时效分，并在条目标注「日期未核实」。

## 4. 融资 / 收购双边确认

- 若 `verify.funding_needs_both_sides=true`：融资、收购类需投资方与被投/被收购方**双边官方口径**都确认。
- 仅单边宣称、对方无回应 → 降级（封顶简讯）或标注「待确认」。

## 5. 反 slop

- 标题去模板化、禁营销词（「震惊」「必看」「绝了」「炸裂」「王炸」等）。
- 摘要讲清「是什么 / 为何重要 / 对读者意味着什么」，不堆形容词。
- 保留 `evidence_quotes` 原文片段留痕，便于复核。

## 6. 终稿自检（与 quality-checklist.md 配合）

收尾前确认：每条必看 ≥2 独立源；每个 metric 带 source_url；所有 `event_date` 在窗口内且已核实；
无任何凭记忆数字；融资类已双边确认。任一不满足 → 降级或删除该条，绝不带病发布。

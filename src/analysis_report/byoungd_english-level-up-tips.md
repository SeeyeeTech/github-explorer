### 问题判断

项目**不是研究出来的，是被一位姑娘 W. 问出来的**（2017 年 7 月，W. 准备 TOEFL 时追问"如何高效学习英语"）。作者在回答过程中意识到三件事：

1. 身边同学在英语学习这件事上磕磕绊绊的人群远比想象中大。
2. "英语应该像母语一样自然习得"是大部分人从没想过的视角。
3. 自己从"高考省第一"到"用力过猛反失败"的反思有资格被系统化分享。

这是"亲密关系触发 → 个人反思 → 公共知识"的典型知识生产链。

### 解法哲学

- **反教条 + 反工具主义**：x-misc 篇直接批"每天 n 分钟英语"是糖衣炮弹；2026 版 AI 篇把"押注一个最强 AI"判为次优。
- **先讲人，再讲方法**：把"认知 / 情绪 / 动机"放在 1-understanding 作为 onboarding gate，技能篇按"输入量门槛从低到高"排（单词 < 听力 < 阅读 < 口语 < 写作）。
- **数据 + 故事双轮**：艾宾浩斯曲线、CEFR、OEC 7000 词阈值给权威感；part-2 "我的故事"（"花 2000 万开公司倒闭"）给人格化与共情入口。
- **作者明确不做什么**：不接赞助、不开通打赏——变现走仓库外的 token.love / ku0.com，"内容人格"与"商业承接"在物理上分离。

## 核心价值提炼

### 创新之处

按新颖度×实用性排序：

1. **认知前置的章节模型**（新颖 4 / 实用 5 / 迁移 5）— 7 章里第一章不讲技能，讲"为什么要学 / 怎么不死 / 怎么不内耗"。把学习科学（动机 / 自我决定理论）渗透进英语方法论的最直接表达。
2. **AI 篇 = 工具链而非工具评测**（新颖 5 / 实用 5 / 迁移 4）— 2026 版 AI 篇明确给出 Gemini / ChatGPT / Claude / Perplexity / DeepL Write 的功能分工（"Gem = 长期课程 / Live = 口语 / Guided Learning = 阅读 / Canvas = 写作 / quizzes = 复习"），并附 2026-03-20 资料更新依据。是"AI 当教学系统"的最实用设计。
3. **"输入 / 输出 / 纠错 / 复习"四步训练回路**（新颖 4 / 实用 5 / 迁移 5）— AI 篇第 10 节把"刻意练习"细化为 5 条可勾选的 weekly template，可直接搬到任何技能训练。
4. **抽象化的视觉记忆反工具主义**（新颖 4 / 实用 4 / 迁移 4）— vocabulary 篇直接批评"图片卡背单词"工具（你记得是图片不是单词），提出"抽象化"图像：脑内的猫，而非某张具体的图。
5. **领域最小词表切片**（新颖 3 / 实用 5 / 迁移 5）— word-list/ 按"语言 / 行业"切 30–70 词小表（Common 70 + Java + VibeCoding 40），比 7000 词大表更接近真实需求。
6. **Issue 当论坛**（新颖 3 / 实用 4 / 迁移 4）— 不开 Discussions，用 issue 自然演化为深度讨论区（#6 单词 App 推荐 62 评论、#3 开放讨论 30 评论），是轻量社区运营模板。
7. **Docsify + 多插件零 build 站点**（新颖 2 / 实用 4 / 迁移 4）— `docs/index.html` 纯 CDN 加载 docsify + themeable + sidebar-collapse + KaTeX + mermaid + markmap + zoom-image + copy-code + back-to-top，无 npm / 无 build / 无 CI。个人纯 markdown 站点的最低维护成本方案。

### 关键设计决策

1. **决策**: 章节编排 = 认知 → 单词 → 听力 → 阅读 → 口语 → 写作 → AI
   - **问题**: 读者带着"应试创伤 + 焦虑"来，第一章不破认知直接讲方法会失败
   - **方案**: 把"该不该学 / 为什么学 / 怎么不内耗"放在 1-understanding 作 onboarding gate；技能篇按"输入量门槛从低到高"排；AI 篇作 2025-2026 增量章节压轴
   - **Trade-off**: 写作被放到靠后（门槛最高）；AI 篇需持续刷新
   - **可迁移性**: **高**。任何"知识 + 方法论"类文档都可借鉴

2. **决策**: 渲染链迁移 GitBook → Docsify → GitHub Pages
   - **问题**: Issue #83 暴露 GitBook 托管服务挂了，自托管必须轻量
   - **方案**: `docs/index.html` 纯 CDN 加载 Docsify + 多插件，零 build step
   - **Trade-off**: 多 CDN 依赖 + KaTeX / mermaid / markmap 几乎未被使用，bundle 有冗余
   - **可迁移性**: **中-高**

3. **决策**: 双语策略 = 中文为母版，英文为镜像子集
   - **问题**: 项目天然有非中文读者（海外华人 / 英文母语者 / Google 搜索来源）
   - **方案**: `docs/en/` 与 `docs/` 同构，README 标 WIP；AI 篇同步率最高
   - **Trade-off**: 英文版完成度 < 中文版，需写完后翻译
   - **可迁移性**: **高**

4. **决策**: 变现走仓库外，内容人格与商业承接物理分离
   - **问题**: 个人 IP 一旦开通打赏，民间方法论就被"利益相关"污染
   - **方案**: README 末段声明"不接受赞助"；变现走 token.love / ku0.com
   - **Trade-off**: 流量变现受限于"道德制高点"约束
   - **可迁移性**: **高**
